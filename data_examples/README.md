# ОПИСАНИЕ ПРИМЕРОВ
В данной папке хранятся используемые в работе примеры в формате JSON-файлов:
1. **example1.json** - Пример из оригинальной статьи
  - 17 кандидатов
  - 2 критерия
  - 4 эксперта
2. **example2.json** - Выбор смартфона - наглядный пример
  - 3 смартфона
  - 2 критерия
  - 3 эксперта
3. **example3.json** - Выбор датасета SQL-to-text - практический пример
  - 6 датасетов
  - 3 критерия
  - 10 экспертов (методов оптимизации)

# СТРУКТУРА JSON ФАЙЛА
Каждый JSON-файл имеет следующую структуру:
```json
{
  "alternatives": ["Альтернатива1", "Альтернатива2", ...],
  "criteria": [
    {
      "name": "Название критерия",
      "type": "positive"  // или "negative"
    }
  ],
  "dms": [
    {
      "id": "УникальныйIDЭксперта",
      "weights": [0.5, 0.5],  // веса критериев (сумма = 1)
      "scores": [              // матрица оценок m×n
        [значение11, значение12, ...],  // для альтернативы 1
        [значение21, значение22, ...],  // для альтернативы 2
        ...
      ]
    }
  ]
}
```

## ТИПЫ КРИТЕРИЕВ
- "positive" - benefit-атрибут (чем больше, тем лучше)
- "negative" - cost-атрибут (чем меньше, тем лучше)

## ПОДРОБНОЕ ОПИСАНИЕ ПОЛЕЙ
1. alternatives (массив строк)
Список всех альтернатив для выбора. Каждая строка — уникальное название альтернативы.

2. criteria (массив объектов)
Характеристики, по которым оцениваются альтернативы.

3. dms (Decision Makers - Эксперты) (массив объектов)
Массив экспертов с их оценками и весами критериев.

## ВАЖНО:
- Количество оценок в scores должно совпадать с количеством альтернатив (m)
- Количество значений в каждом массиве scores[i] должно совпадать с количеством критериев (n)
   Сумма весов в weights должна быть равна 1
